# ansible-playbook-pssid-daemon

This is a set of Ansible Playbooks for deploying Raspberry Pi probes that run [pSSID](https://github.com/UMNET-perfSONAR/pSSID). The playbooks should be run against inventories generated by [pSSID-GUI](https://github.com/UMNET-perfSONAR/pSSID-GUI).

## Clone this playbook

```
git clone https://github.com/UMNET-perfSONAR/ansible-playbook-pssid-daemon.git
cd ansible-playbook-pssid-daemon
```

## Getting Dependencies

```
ansible-galaxy install -r requirements.yml
```

## Usage

### Installing pSSID on Probe(s)

One option is to specify a particular IP address to deploy to (note the trailing comma):

```
ansible-playbook playbooks/install-pssid.yml -i 1.1.1.1, --ask-become-pass
```

Another option is to deploy to all IP addresses in an inventory generated by pSSID-GUI:

```
ansible-playbook playbooks/install-pssid.yml -i /path/to/pssid-gui-inventories/inv-name/inventory --ask-become-pass
```

### Pushing pSSID Configurations to Probes

```
ansible-playbook playbooks/push-configs.yml -i /path/to/pssid-gui-inventories/inv-name/inventory --ask-become-pass
```

### Pushing SSIDs and Password to Probes

The setting of SSIDs and their corresponding passwords is done by pushing an encrypted JSON file (`templates/logins.json`) to the probes that is then used by [pScheduler](https://github.com/perfsonar/pscheduler).

Create and encrypt the secrets file:

```
cp templates/logins.json.example templates/logins.json
vi templates/logins.json
ansible-vault encrypt templates.json
```

Then push the secrets to all probes in the inventory:

```
ansible-playbooks playbooks/push-secrets.yml -i /path/to/pssid-gui-inventories/inv-name/inventory --ask-become-pass --ask-vault-pass
```

### Generating and Saving pSSID Configurations Locally

```
ansible-playbook playbooks/create-local-configs.yml -i /path/to/pssid-gui-inventories/inv-name/inventory
```

The configs will be saved within `/path/to/pssid-gui-inventories/inv-name/configs` locally.

### Pull Identifiers From Probes

```
ansible-playbook playbooks/pull-identifiers.yml -i /path/to/pssid-gui-inventories/inv-name/inventory --ask-becom-pass
```

The identifiers (e.g. MAC address) are saved within `/path/to/pssid-gui-inventories/inv-name/serial` locally.
