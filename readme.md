# ansible-playbook-pssid-daemon

Ansible playbook for deploying Raspberry Pi probes that run [pSSID](https://github.com/UMNET-perfSONAR/pSSID). This playbook should be run against inventories generated by [pSSID-GUI](https://github.com/UMNET-perfSONAR/pSSID-GUI).

## Dependencies

Run the following to get dependencies:

```
ansible-galaxy install -r requirements.yml
```

## Usage

### Secrets

`logins.json` is a file encrypted by Ansible that will be copied to the target Raspberry Pi and contains login credentials in the format required by [pScheduler](https://github.com/perfsonar/pscheduler)'s SSID test. Use the [`ansible-vault`](https://docs.ansible.com/ansible/latest/cli/ansible-vault.html) command to manage the file.

### Creating the Inventory

Using pSSID-GUI, create an inventory which describes the configuration of pSSID.

### Running

```
ansible-playbook playbook.yml --become --become-user root --ask-become-pass --become-method su --ask-vault-pass -i ../pSSID-GUI/inventories/$name/inventory
```

Modify `../pSSID-GUI/inventories/$name/inventory` to be a path to an inventory generated by pSSID-GUI that should be run.
